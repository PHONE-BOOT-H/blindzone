# BlindZone 발표 outline (초안)

> 외부 AI 2차 평가(2026-05-23) 반영본. XGBoost를 발표 중심이 아닌 4순위 설명 보조로 내리고, **robust blind zone**을 핵심 메시지로 세웠다.
>
> **이건 초안이다. 발표 톤·강조점·슬라이드 분량은 한태영 본인 판단으로 재구성할 것.** 수치는 모두 검증된 실제값(`weight-sensitivity.md` 참고).

## 핵심 메시지 (한 줄)

사고다발지도가 놓치는 응급 사각지대를, **가중치를 바꿔도 반복적으로 드러나는(robust)** 방식으로 발굴한다.

## 왜 이 프레임인가 (외부 평가 핵심)

위험지수는 0.4/0.3/0.3 가중합 정의이므로 "왜 그 가중치냐"가 최대 약점이다. 단일 순위를 정답처럼 방어하지 말고, **여러 합리적 가중치에서 반복 상위에 남는 지역**으로 메시지를 옮긴다. 그러면 임의 가중치가 "관리된 불확실성"으로 바뀐다.

## 발표 흐름 (제품/서비스 데모 중심)

### 1. 문제 — 사고지도의 사각지대
- 한국 골든타임 대응 부족 (머니S 2025-04-10: 선진국 대비 사망률 2배, 사고 후 1시간 내 수술실 도착률 50%)
- 기존 사고다발지도가 보여주는 "사고 자주 난 곳"은 대부분 대도시 — 이미 모두가 안다.
- **질문**: 사고는 드물지만, 한 번 나면 응급 대응이 구조적으로 불가능한 곳은 어디인가?

### 2. 데이터 — 공공데이터 4종 융합 (전국 시군구 252개)
- TAAS 사고다발지역 × 응급의료기관 × 119 구급통계 × 행정경계
- → 데이터 융합 가점 근거

### 3. 방법 — 4단계 (XGBoost는 맨 뒤 보조)
1. **가중지수**: 사고빈도·사망률·응급접근성을 min-max 정규화 후 가중합 (기준 0.4/0.3/0.3)
2. **가중치 민감도 검증** ← 핵심. 126개 시나리오(명시 6 + 격자 120) + 사망률 소표본 보정
3. **What-if 시뮬레이션**: 가상 응급거점 추가 시 거리 기반 접근성 변화 (정책효과 예측이 아니라 거리 민감도)
4. **설명 보조**: XGBoost surrogate + SHAP으로 "왜 위험한가" 요인 제시 (예측 모델 아님)

> XGBoost 성능(R²)·AI 도구 활용 내역은 발표 중심에 두지 않는다. "AI로 포장했다"는 인상을 피하고, 메인 스토리는 문제·데이터·발견·데모로 간다. AI 가점 근거는 별도 슬라이드/부록으로.

### 4. 발견 — robust blind zone (필수 핵심 슬라이드)

사고건수 하위인데도 가중치 전반에서 상위에 남는 곳은 **인제·옹진 단 두 곳**이다.

| 시군구 | 사고건수 | 기준 순위 | 중앙값 순위 | top10 등장 비율 | 성격 |
|---|---:|---:|---:|---:|---|
| 인제군 | 1 | 2 | 2 | 0.905 | 드물지만 치명적 (단 소표본 사망률 주의) |
| 옹진군 | 0 | 4 | 3 | 0.730 | 도서 구조적 사각지대 (보정에도 강건) |
| (참고) 송파·동대문 등 6곳 | 1,200~2,000+ | 1~8 | — | 0.64~0.87 | 대도시 사고다발 — 사각지대 아님 |

- robust 8곳 중 6곳은 대도시. **비대도시 robust는 인제·옹진뿐** = 우리가 찾는 사각지대.
- **옹진군**: 응급거리 75.3km 단독으로, 가중치·사망률 보정 양쪽에 강건 → 가장 안정적인 대표 사례.
- **인제군**: 기준 2위지만 사망률 1.0(사고 1건의 산물)에 의존 → 보정 시 15위로 하락. **정직하게 고지**하되, 거리 24.2km라는 구조적 요인은 남음.

### 5. 데모 (라이브 또는 영상)
- 시민 모드: 시군구 선택 → 위험지수 + SHAP "왜 위험한가"
- 정책 시뮬레이터: 가상 거점 추가 → 접근성 변화 (인제군 거리 24.2 → 8.2km, 위험지수 -58%)
- About: 발견 + 한계 고지

### 6. 정직성 — 차별점이자 신뢰의 근거
- 모델이 하지 않는 것 명시 (사고 예측 X / 사망 예측 X / 정책효과 예측 X)
- 가중치 임의성 → 민감도 분석으로 방어
- 인제 소표본 사망률 → 숨기지 않고 보정 결과 병기
- 직선거리·다발지점 한정·수요변수 미포함 한계 고지

### 7. 가점 신청 (부여는 심사위원 판단)
- AI 학습도구: Claude Code 코딩 보조
- AI 분석도구: XGBoost + SHAP
- 데이터 융합: 공공데이터 4종 시군구 결합

## 예상 질문 대응 스크립트

- **"왜 0.4/0.3/0.3?"** → "임의 설정입니다. 그래서 126개 가중치 시나리오로 검증했고, 단일 순위가 아니라 어떤 가중치에서도 robust하게 남는 지역을 봅니다."
- **"가중합인데 XGBoost를 왜?"** → "예측이 아니라 SHAP 기반 설명과 시뮬레이션용 surrogate입니다. 위험지수 본체는 가중합입니다."
- **"직선거리 한계는?"** → "실제 이송시간이 아니라 접근성 하한값입니다. 도로망·도서·산악 미반영을 인정하며, What-if도 정책효과가 아니라 거리 민감도입니다. 덧붙여, 거리에 균일 배수를 곱해도 순위는 불변이고(min-max), 외진 곳 우회를 가중 보정해도 인제·옹진은 상위를 유지합니다(weight-sensitivity 6.1)."
- **"인제 2위 믿을 수 있나?"** → "사망률 1.0은 사고 1건의 산물이라 보정하면 15위로 내려갑니다. 그래서 사망률에 의존하지 않는 옹진군을 더 robust한 대표 사례로 제시합니다."

## 선행 연구 포지셔닝

- Jung & Qin (2024 MDPI, 2025 Sage TRR): 한국 EMS+사고 융합, XGBoost+SHAP 선행이 존재 → 인지하고 인용.
- BlindZone 차별점: 시군구 252개 전국 단위 + 4종 동시 융합 + What-if 시뮬레이션 + 풀스택 웹서비스 + robust 검증.
