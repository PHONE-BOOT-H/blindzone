# 시제품 포트폴리오 — BlindZone

> 2026 국토교통 데이터 활용 경진대회 / 제품·서비스 개발 분야 별첨.
> 심사 중 백엔드 sleep·로딩 지연 대비 캡처(보험). 모든 화면은 실제 라이브에서 캡처.

- 라이브 서비스: https://blindzone-brown.vercel.app
- 백엔드 API: https://hananhan-blindzone-backend.hf.space
- 소스코드: https://github.com/PHONE-BOOT-H/blindzone

## 화면 1 — 메인: 전국 위험지도 + TOP10
**파일: 10_demo_main.png**
전국 252개 시군구의 잠재 위험지수를 지도에 표시. 상위는 대부분 대도시(이미 알려진
사고다발지). 하단 TOP10 목록과 함께, 사고 빈도 지도가 보여주는 그림을 먼저 제시한다.

## 화면 2 — 시군구 상세 + "왜 위험한가"(SHAP)
**파일: 11_demo_detail.png** (옹진군 선택)
시군구를 선택하면 위험지수·사고 건수·응급기관 거리·도착시간을 보여주고, AI(SHAP)가
**왜 위험한지** 기여요인으로 설명한다. 옹진군은 다발지점 사고 0건이나 응급거리 75.3km로
응급 접근성이 위험의 핵심 요인임을 확인할 수 있다. (사각지대 대조·TOP10도 함께 표시)

## 화면 3 — 정책 시뮬레이터
**파일: 12_demo_policy.png**
가상의 응급의료 거점 좌표를 추가하면, 전국 시군구의 최근접 거리가 재계산되고 위험지수가
어떻게 변하는지 즉시 보여준다. 평균 위험지수 변화·최대 감소·개선 시군구 수를 함께 제시
(정책효과 예측이 아니라 거리 기반 접근성 민감도 분석).

## 화면 4 — 서비스 소개(About)
**파일: 13_demo_about.png**
무엇을 푸는지·어떻게 찾는지·믿을 수 있는 근거(다섯 가지 검증)·한계를 일반 사용자도
이해할 수 있게 정리.

## 기술 구성
- 프론트엔드: Next.js 14 + TypeScript + MapLibre GL + deck.gl (Vercel 배포)
- 백엔드: FastAPI + XGBoost + SHAP (Hugging Face Spaces, Docker)
- 안정성: 백엔드 일시 중단 시에도 정적 스냅샷으로 지도·위험지수·기여요인·대조가 동작
