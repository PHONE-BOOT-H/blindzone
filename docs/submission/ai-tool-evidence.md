## 1. AI 학습도구 — Claude Code

### 활용 도구

- **Claude Code** (Anthropic, CLI 기반 코딩 보조 에이전트)
- 활용 기간: 2026-05-19 ~ 2026-05-23 (5일)

### 활용 영역

| 영역 | Claude Code 보조 | 한태영 직접 판단·결정 |

| 데이터 파이프라인 | 시군구 좌표/거리/집계 함수 코드 작성, pytest 테스트 추가 | 데이터셋 선택, 시군구 단위 join 키 결정, EPSG 좌표계 선택 |

### 한태영의 핵심 판단 영역 (AI 보조 X)

- **프로젝트 컨셉**: "사고 빈도 + 사망률 + 응급 접근성 결합" 아이디어 자체
- **데이터셋 선택**: TAAS, 응급의료기관(B552657), 119 구급통계(1661000), 통계청 (센서스경계) — 4종 융합 결정
- **위험지수 정의**: min-max 정규화 후 가중합(0.4/0.3/0.3)으로 단순화하기로 결정
- **라이선스 검토**: VWORLD `dsId=30604` (공공누리 4유형, 상업 X·변경 X) → 통계청 (센서스경계) VWORLD `dsId=10825` (CC BY-NC-ND, 비영리 출품 가능)로 변경 결정
- **외부 평가 반영**: 별도 AI에 1차 평가 의뢰 → 위험표현 8개 정정 + 사각지대 발굴(`/api/contrast`) 신설 우선순위 결정
- **차별점 정의**: 인제·옹진 outlier 발굴 narrative — 사고건수 순위 vs BlindZone 순위 격차가 큰 시군구를 대표 사례로 선정
- **풀스택 전환**: 초기 Streamlit → Next.js 14 + FastAPI로 아키텍처 재설계 결정

### 활용 규모 (참고치)

- **총 commit 수**: 53건 (Phase 1 데이터 셋업 ~ Phase D 제출 자료)

- Claude Code는 **코드·문서의 초안을 빠르게 작성하는 도구**로 활용했으며, 모든 출력은 한태영이 검토·수정 후 commit했다.

## 2. AI 분석도구 — XGBoost + SHAP

### 사용 모델

- **XGBoost** (`xgboost.XGBRegressor`)
  - 정의된 잠재 위험지수(가중합)를 입력 변수(사고 빈도, 사망사고 비율, 응급기관 거리, 응급 도착 추정시간, 면적)로 재학습한 **surrogate 회귀 모델**
  - 사고·사망 예측 모델이 아니라, **정의식의 비선형 surrogate** — R²는 사고 예측 성능이 아니라 정의식 재현도
  - 자세한 모델 카드: [model_card.md](model_card.md)

- **SHAP TreeExplainer** (`shap.TreeExplainer`)
  - 학습된 surrogate 모델에 대해 시군구별 상위 기여 요인(위험 증가/감소 방향 포함) 추출
  - 결과는 `feature_details.parquet`에 사전 계산 → API 런타임 비용 0
  - 시민 모드 페이지에서 시군구 선택 시 "왜 위험한가" 섹션으로 노출

### 사용 근거

- 가중합 위험지수는 단순 정의식이지만, 시군구별로 **어느 요인이 위험을 끌어올리는지** 직관적으로 보여주려면 SHAP 기여도 분해가 필요
- 인제군 사례: SHAP top1 = 응급기관까지 거리(+0.173), top2 = 사고 빈도(+0.072), top3 = 사망사고 비율(+0.065) — 응급 접근성이 단일 요인으로 가장 큰 기여 ([case-study.md](case-study.md))

## 3. 데이터 융합 — 공공데이터 4종 결합

| 출처 | 형식 | 시군구 단위 변환 |
