# BlindZone 기획서 본문 초안 (HWP 붙여넣기용)

> 통계누리 HWP 양식에 붙여넣을 본문 텍스트 초안. 외부 평가 권고 구조(첫 2페이지에 문제·비교·발견·검증·제품·한계). 표현은 검증된 사실만. **확정 위험 판정이 아니라 현장검증 우선순위**로 일관 서술.
> 라이브: 프론트 https://blindzone-brown.vercel.app · 백엔드 https://hananhan-blindzone-backend.hf.space · 코드 https://github.com/PHONE-BOOT-H/blindzone

---

## 1. 작품 개요

**BlindZone — 사고다발지도가 놓치는 응급 사각지대를 찾는 전국 시군구 위험지수 웹서비스**

교통사고 위험지도는 늘 대도시를 가장 진하게 보여준다. 사고가 많기 때문이다. 그러나 진짜 물어야 할 것은 "사고가 자주 나는 곳"이 아니라 **"사고가 났을 때 제때 병원에 도착하기 어려운 곳"**이다. BlindZone은 교통사고 다발지점·사망사고 비율·응급의료기관 접근성을 전국 252개 시군구 단위로 융합해, 기존 사고다발지도가 놓치는 **응급 사각지대 후보**를 발굴하고 정책 시뮬레이션을 제공하는 풀스택 웹서비스다.

## 2. 문제 정의

- 한국은 교통사고 후 골든타임 대응 취약이 지적되어 왔다(보도: 선진국 대비 사망률, 사고 후 1시간 내 수술실 도착률 등).
- 기존 사고다발지도·교통안전 정책은 **사고 빈도 중심**이라 대도시 반복사고에 강하지만, 농어촌·도서의 분산 사고와 응급 접근성 문제를 놓친다.
- 모든 지역을 정밀조사할 수는 없다. **어디를 먼저 봐야 하는가(현장검증 우선순위)**를 데이터로 선별하는 도구가 필요하다.

## 3. 데이터와 방법

**위험지수 — 공공데이터 3종 융합(전국 시군구 252개)**
- TAAS 사고다발지역(한국도로교통공단): 사고 빈도·사망사고 비율
- 응급의료기관 정보(국립중앙의료원): 시군구 중심 → 최근접 응급기관 거리
- 시군구 행정경계(통계청 센서스경계): 공간 결합 기준

**방법**
1. min-max 정규화 후 가중합(0.4/0.3/0.3)으로 **잠재 위험지수** 정의 — 투명한 공식
2. XGBoost surrogate + SHAP으로 지역별 기여요인을 **설명**(예측 모델이 아닌 설명 레이어)
3. What-if 시뮬레이션: 가상 응급거점 추가 → 거리 재계산 → 위험지수 변화(접근성 민감도)

> 본 서비스는 사고·사망을 직접 예측하지 않는다. 정의된 지수를 계산·설명하는 도구이며, 결과는 확정 판정이 아니라 현장검증 우선순위다.

## 4. 핵심 발견 — robust blind zone

위험지수 상위는 대부분 대도시(이미 알려진 사고다발지)다. 126개 가중치 시나리오에서 **반복적으로 상위에 남는 비대도시는 인제군·옹진군 두 곳뿐**이다.

| 시군구 | 다발지점 사고 | 전체 사고 | 응급거리 | 고령비율 | 성격 |
|---|---:|---:|---:|---:|---|
| 옹진군(인천·도서) | 0건 | 27건(사망2) | 75.3km(전국 최대) | 37.2% | 거리 단독·가장 안정적 |
| 인제군(강원·산간) | 1건 | 83건(사망4) | 24.2km | 28.0% | 거리+사망률(소표본 주의) |

- "사고 0~1건"은 **다발지점 기준**일 뿐, 전체 교통사고는 인제 83·옹진 27건이다. 다발지점은 이를 1.2%·0%만 포착한다(대도시는 80%+ 포착) — 다발지점 지도가 분산형 사각지대를 놓친다는 직접 증거.
- 두 곳 다 고령비율이 전국 평균의 약 1.5배다. **사고가 적은 게 아니라, 응급 수요는 큰데 접근이 구조적으로 막힌 곳.**

## 5. 신뢰의 근거 — 다섯 가지 검증

"직접 정의한 지수가 우연 아니냐"에 대한 답. 인제·옹진은 다섯 축 모두에서 사각지대로 확인된다.
1. **가중치 민감도** — 126개 시나리오에서 비대도시 robust는 인제·옹진뿐
2. **사망률 소표본 보정** — 인제 의존성 정직 고지(옹진은 사망률 무관)
3. **도로망 실거리(OSRM)** — 252개 직선→실거리 재산출해도 결론 불변(실제 도로 인제 48.7분·옹진 183.5분)
4. **정부 응급의료취약지** — 도달시간+인구 기반 정부 공식 지정과 같은 곳 지목(수렴 타당도)
5. **다발지점 한계 대조** — 전체 사고 통계로 다발지점의 과소포착 직접 확인

## 6. 제품/서비스 (라이브 시연)

- **시민 모드**: 지도에서 시군구 클릭 → 위험지수 + SHAP "왜 위험한가" 설명
- **정책 시뮬레이터**: 가상 응급거점 추가 → 접근성·위험지수 변화 즉시 확인(거점 1곳 추가 시 인제 30,773명·옹진 19,397명 접근성 개선 잠재)
- **사각지대 대조**: 사고건수 순위 vs 위험지수 순위 격차 시각화
- **기술**: Next.js 14 + MapLibre/deck.gl(프론트, Vercel) / FastAPI + XGBoost + SHAP(백엔드, HF Spaces). 백엔드 다운 시에도 정적 스냅샷으로 핵심 데모가 동작하도록 설계.

## 7. 기대효과·활용

- 응급의료 자원(거점·헬기·이송체계) 배치 검토 시 **우선 현장검증 대상**을 객관 지표로 선별.
- 사고 빈도 중심 정책의 사각을 보완 — "사고 많은 곳"에서 "사고 나면 늦는 곳"까지 시야 확장.
- 시군구 연속 지수·기여요인·시뮬레이션 제공으로, 정부 취약지 이분 지정(지정/비지정)보다 풍부한 의사결정 보조.

## 8. 차별성 (선행 인지)

한국 데이터로 사고·응급의료를 결합한 선행(Jung & Qin 2024·2025, XGBoost+SHAP)이 존재한다. BlindZone은 이를 인지하고 **확장**한다: 전국 시군구 단위 + 사각지대 발굴 narrative + 인터랙티브 What-if + 풀스택 웹서비스 + 5중 검증. (방법론 자체의 최초성을 주장하지 않으며, 문제의식과 제품화·전국화가 차별점.)

## 9. 가점 신청 (부여는 심사위원 판단)

- **AI 학습도구**: Claude Code 코딩 보조
- **AI 분석도구**: XGBoost 회귀 + SHAP TreeExplainer(지수 설명 레이어)
- **데이터 융합**: 3종(TAAS×응급의료기관×행정경계) 시군구 융합 + 119 구급통계 교차검증

## 10. 한계 및 정직 고지

- 사고·사망을 예측하지 않는다(정의식 surrogate). 결과는 현장검증 우선순위.
- 응급 도착시간은 거리 기반 추정(실제 출동·병원 수용·이송체계 미반영). 옹진 도서는 거리 지표가 실제 해상 이송을 과소추정.
- 가중치는 실증 근거 없는 선택 → 126 시나리오로 검증. 시군구 단위 평균(V2에서 격자 세분화).
