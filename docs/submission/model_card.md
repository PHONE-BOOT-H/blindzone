# Model Card — BlindZone XGBoost Risk Surrogate

> 본 문서는 BlindZone의 모델 사용 의도·한계·해석 가이드를 명시한다.
> 절대 원칙: 모델이 하지 않는 것은 모델이 한다고 주장하지 않는다.

## 1. 모델 목적

**본 모델은 실제 교통사고 발생이나 사망률을 직접 예측하는 모델이 아니라**, 공개 데이터로 정의한 `잠재 위험 지수(risk_index)`를 설명하고 시뮬레이션하기 위한 **탐색형 surrogate model**이다.

- `risk_index` 정의: `data_manifest.md` 참고. 정규화된 사고 빈도·사망사고 비율·응급 접근성 변수의 가중합 (0.4 / 0.3 / 0.3).
- 모델 역할: 이 정의식의 출력을 XGBoost로 재학습 → SHAP 값으로 시군구별 "왜 위험한가" 상위 요인 추출 + 가상 응급거점 추가 시 inference loop으로 위험 변화 시뮬레이션.

즉, **"위험 예측 모델"이 아닌 "위험 지수 설명·시뮬레이션 wrapper"**.

## 2. 모델 입력 / 출력

**입력 피처 (7개)**:
- `accident_count` — 시군구 내 사고 다발지점 사고 건수 합 (TAAS 다발지역 데이터 기준, 전체 교통사고 X)
- `fatality_count` — 사망사고 수
- `fatality_rate` — 사망사고 / 전체사고
- `injury_count` — 부상신고자 수
- `ems_distance_km` — 시군구 중심점에서 최근접 응급의료기관까지 거리
- `ems_response_min` — `ems_distance_km / 60 × 60` (60km/h 가정, 추정값)
- `area_km2` — 시군구 면적

**출력**: `risk_index` (실수, 0~1 범위 근방)

## 3. 학습 데이터

- 입력: 250+개 시군구 (시군구 경계 SHP 기준, BASE_DATE 2025-06-30)
- 데이터 양: 250+ 샘플. 머신러닝 관점으로는 매우 작은 데이터. 학습이라기보다 **함수 fit** 성격.
- 학습/평가 분할: 80/20 random_state=42

## 4. 평가 지표 (정확한 해석)

| 지표 | 값 | 정확한 의미 |
|---|---|---|
| R² | 0.90 | XGBoost가 위에 정의된 가중합 risk_index를 **얼마나 잘 재현**하는지. **실제 사고 예측 성능 아님**. |
| MAE | 0.0079 | 동일 지표에 대한 평균 절대 오차. |

R²가 높은 이유는 risk_index가 입력 피처의 가중합 정의식이라 자연스러운 결과 (모델이 사실상 그 함수를 학습). **R²만 보고 "사고 예측 정확도 90%"라고 해석하면 잘못된 주장**.

## 5. 모델이 하지 않는 것

다음은 본 모델이 **하지 않는다**. 발표·기획서·About 페이지·README에서 절대 주장하면 안 된다.

- ❌ 실제 사고 발생 확률을 예측한다
- ❌ 사망률을 예측한다
- ❌ 응급 도착 시간을 실시간 예측한다 (실제 119 출동 raw 데이터 없음)
- ❌ 응급기관 추가의 **정책 효과를 예측**한다 (거리 기반 접근성 지표 변화일 뿐)
- ❌ 일반인용 사고 다발지도가 부재한 시장 공백을 100% 검증한다 (시장 조사 가설)

## 6. 모델이 하는 것

- ✅ 정의된 잠재 위험 지수에 대한 surrogate model을 제공한다 (재현·해석 가능성)
- ✅ SHAP 값으로 시군구별 "어떤 피처가 위험에 가장 기여하는지" 상위 요인을 제시한다
- ✅ 가상 응급거점 추가 시 응급기관 거리 피처가 어떻게 변하고, 그에 따라 위험 지수가 어떻게 재계산되는지 보여준다 (민감도 분석)
- ✅ 공공데이터 4종을 시군구 단위로 융합한 결과를 인터랙티브 지도·표로 탐색하게 한다

## 7. 한계

1. **소규모 데이터**: 250+ 시군구는 ML 데이터로 매우 작음. XGBoost 같은 강력 모델이 단순 가중합을 거의 완벽히 재현하는 것은 모델 성능이 아닌 데이터·정의식 구조의 결과.
2. **데이터 한계**:
   - TAAS 다발지역 = 전체 사고 X (사고 다발 지점만)
   - 119 raw 비공개 → 응급 도착시간은 거리 기반 추정 (60km/h 가정)
   - 시군구 단위 평균 → 격자 내 변동성 평준화 (V1.1에서 1km 격자 검토)
3. **가중치 임의 설정**: 0.4 / 0.3 / 0.3은 실증 근거 없이 선택. 다른 가중치 조합 시 결과 크게 달라질 수 있음 (가중치 민감도는 데모에 포함 가능).
4. **SHAP의 의미**: 정책 효과 인과 추론이 아니라 정의된 지표의 변수 기여도. SHAP 값을 "정책 개입 효과"로 해석 X.

## 8. 사용 의도

- 정책 결정자에게 **후보 지역 탐색 도구**로 제공
- 시민에게 **우리 동네 응급 사각지대 지표** 안내
- 심사·연구자에게 **데이터 융합 방법론 시연**

**의사결정 직접 대체 X**. 추가 검증·전문가 판단 필수.

## 9. 모델 저장·재현

- 모델 파일: `backend/models/xgb_risk_model.pkl` (pickle, model + feature_cols + metrics)
- 학습 스크립트: `backend/src/train.py`
- 학습 명령: `python -m src.train`
- 재현: `data_manifest.md`의 데이터 절차로 grid_features.parquet 재생성 후 학습

## 10. 책임 고지

본 모델 결과는 분석·탐색 목적의 시제품 산출물이며, 실제 정책·의료·교통 의사결정의 단독 근거로 사용해서는 안 된다. 대회 출품용으로 작성됨.
