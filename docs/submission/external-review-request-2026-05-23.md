## 0. 프로젝트 한 줄

**사고 빈도 지도가 놓치는 응급 사각지대를, 시군구 단위 4종 공공데이터 융합으로 발굴하는 탐색형 위험 지수 + What-if 시뮬레이터 (FastAPI + Next.js 풀스택 웹서비스).**

코드네임: BlindZone.

## 1. 무엇을 푸는가

한국 교통사고 골든타임 대응 부족이 보도·연구에서 지적되어 왔다 (예: 머니S 2025-04-10, "선진국 대비 사망률 2배"·"사고 후 1시간 내 수술실 도착률 50%"). 기존 사고 다발지도는 사고 빈도 위주라는 점에 주목해, BlindZone은 **사고 빈도·사망사고 비율·응급의료 접근성을 결합한 시군구 단위 탐색형 위험 지수**를 산출하고, 가상 응급의료 거점 추가 시 거리 기반 접근성이 어떻게 변하는지 보여주는 surrogate 도구다.

본 프로젝트는 **사고를 직접 예측하는 모델이 아니다.** 공개 데이터로 정의한 잠재 위험지수를 계산·설명하는 surrogate.

## 2. 핵심 발견 — 사고 빈도만으로 안 보이는 사각지대

본 데이터로 산출한 위험지수 TOP10 중 **3곳(인제·옹진·수성)** 은 TAAS 사고다발지역 데이터의 사고건수 TOP10에 포함되지 않는다.

### 대표 사례 — 인제군 (강원도)

| 항목 | 값 |
|---|---|
| TAAS 다발지점 사고 건수 | **1건 (전국 245위)** |
| 사망자 | 1명 (사망사고 비율 1.0 — 표본 1건의 한계 명시) |
| 응급의료기관 거리 | **24.20 km** |
| 응급 도착 추정시간 | 24.2분 (직선거리 × 60km/h 가정) |
| **BlindZone 위험지수** | **0.396 (전국 2위)** |

**SHAP top3 기여요인** (왜 위험한가):
1. 응급기관까지 거리 +0.173 (위험 증가) ← 단일 요인 최대 기여
2. 사고 빈도 +0.072
3. 사망사고 비율 +0.065

**가상 EMS 추가 시뮬레이션** (인제 인근 128.17°E, 38.06°N에 거점 1곳 추가):
- 인제군 EMS 거리: 24.20 km → **8.23 km** (-15.97 km)
- 인제군 위험지수: 0.396 → **0.166** (-0.230, 약 **-58%**)
- 인근 47개 시군구도 동시 개선

### 보조 사례 — 옹진군 (인천 도서 지역)

TAAS 다발지점 사고 0건 (전국 246위) → 위험지수 4위. 도서 지역 응급 접근성 한계의 누적 결과.

## 4. 방법론

1. **시군구 단위 변수 산출** — 사고 빈도(TAAS 다발지역 내), 사망사고 비율, 응급의료기관 거리(km), 응급 도착 추정 시간(거리×60km/h), 면적
2. **잠재 위험지수 정의** — min-max 정규화 후 가중합 (0.4 / 0.3 / 0.3). **가중치는 실증 근거 없는 임의 설정 (명시 고지)**
3. **XGBoost surrogate 회귀** — 정의된 위험지수를 입력 변수로 재학습. **사고·사망 예측 X, 정의식의 비선형 surrogate**. R²는 사고 예측 성능이 아니라 정의식 재현도.
4. **SHAP TreeExplainer** — 시군구별 상위 기여 요인 + 방향 추출. 사전 계산해 API 런타임 부담 0.
5. **What-if 시뮬레이션** — 가상 응급의료 거점 좌표 입력 → 모든 시군구의 거리 피처 재계산 → 동일 모델 inference → 위험 지수 변화. **정책 효과 예측 X, 거리 기반 접근성 민감도 분석 (명시 고지)**

## 5. 차별점 5개 (prior-art 조사 후 확인된 unique 영역)

본 프로젝트는 외부 AI에 의한 선행 연구 조사 (2026-05-23)를 거쳤다. 가장 가까운 선행은 **Jung & Qin (2024 MDPI Sustainability, 2025 Sage TRR)** 한국 EMS 인프라 + 사고 데이터 융합 연구. **그러나 다음 5개의 결합은 검색 범위 내 동등 사례 미발견**:

1. **시군구 252개 전국 단위** 잠재 위험지수 산출 — 선행은 점단위/고속도로/광역시 한정
2. **4종 데이터 동시 융합** (TAAS + 응급의료기관 + 119 구급통계 + 행정경계) — 선행은 보통 2-3종
3. **가중합 위험지수 + XGBoost surrogate + SHAP 기여요인** 결합 파이프라인 — 가중합 → ML surrogate화는 한국 교통 도메인에서 드묾
4. **인터랙티브 What-if 시뮬레이션** (가상 거점 → 거리 재계산 → 동일 모델 재추론) — 학습된 ML 모델에 피처 재주입하는 정책 시나리오 평가 방식
5. **풀스택 웹서비스** (MapLibre + deck.gl + FastAPI) — 선행은 모두 학술 분석 산출물

- min-max + 가중합 위험지수는 환경/안전 평가 표준 기법
- 응급의료 접근성 직선/네트워크거리 계산은 다수 선행

### 가장 가까운 선행 — Jung & Qin 시리즈 (명시 인용 권장 대상)

## 6. 한계 및 정직 고지 (외부 평가 1차 반영 + 추가 분석)

### 위험 표현 정정 8건 (1차 외부 평가 반영 완료)

| 정정 전 (위험) | 정정 후 |
|---|---|
| "사고 위험을 예측한다" | "공개 데이터로 정의한 잠재 위험지수를 계산하고 설명한다" |
| "응급 도착시간" | "응급의료기관 거리 기반 추정 접근시간 (60km/h 가정)" |
| "정책 시뮬레이터가 응급기관 추가 효과를 예측한다" | "가상 응급거점 추가 시 거리 기반 접근성 지표 변화 민감도 분석" |
| "연간 사고 건수" | "TAAS 사고다발지역 데이터 내 사고 건수" |
| "기존 일반인용 서비스는 부재했다" | "기존 사고다발지역 정보는 빈도 중심, 본 프로젝트는 응급 접근성을 결합한 탐색" |
| "주관기관 4종 결합" | "공공데이터 4종 결합" |
| "AI 가점 15점 확보" | "가점 신청 (부여는 심사위원단 판단)" |
| "R² > 0.5 = 모델 성능" | "정의된 잠재 위험지수에 대한 surrogate model 재현도" |

### 본 프로젝트의 한계 (정직 고지)

- 본 모델은 실제 사고·사망을 예측하지 않는다. 정의된 위험지수에 대한 surrogate model이며, R²는 사고 예측 성능이 아니라 정의식 재현도다.
- TAAS 사고다발지역 데이터는 전체 사고 X — 다발지점 한정.
- 119 출동 사건별 raw 비공개 → 응급 도착시간은 응급기관 직선거리 × 60km/h 가정 추정. **도로망·실제 응급차 속도 미반영.**
- 시군구 단위 평균 — 격자 내 변동성 평준화.
- 가중치 0.4 / 0.3 / 0.3은 실증 근거 없는 임의 설정. 다른 가중치 조합 시 결과 달라질 수 있음.
- 인구밀도·고령자 비율 등 수요측 변수 미포함.
- 인제군 사망사고 비율 1.0은 다발지점 1건 중 1건 사망 — 표본 매우 작아 통계적 일반화 어려움.
