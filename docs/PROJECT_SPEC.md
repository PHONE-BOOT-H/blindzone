## 문제 정의

사고 후 1시간 내 수술실 도착률이 절반에 그치고 한국 교통사고 골든타임 대응이 부족하다는 점이 보도·연구에서 지적되어 왔다(머니S 2025-04-10 기사 등). 그러나 기존 사고 다발지도는 단순 빈도만 보여줌. **"사고 위험 × 응급 도달 시간"**을 결합해 잠재 위험지대를 발굴하는 일반인용 서비스는 부재.

진짜 위험한 곳은 **사고는 보통인데 응급차가 늦어 사망률 높은 곳** — 통계가 가려둔 잠재 위험지대.

## 목표 사용자 / 수혜자

- **일반 시민** (메인) — 우리 동네 응급 사각지대 확인
- **정책결정자/지자체** (보조) — 응급 자원 추가 배치 시뮬레이션
- **심사위원/연구자** (간접) — 발견형 인사이트·정책 implication 평가

단일 웹앱에 두 페이지(시민 / 정책)로 통합. 별도 About 페이지.

## 핵심 기능 / 산출물

**시민 모드**
1. 시군구 검색 → 위험지수, 응급도착시간, "왜 위험한가" SHAP 설명
2. 전국 위험지도 (시군구별 색·반경, 호버 툴팁)
3. 상위 N개 잠재 위험지대 리스트

**정책결정자 모드**
4. 가상 응급기관 위치 추가 시뮬레이션 (좌표 입력 또는 지도 클릭)
5. 영향 정량 평가 (전국 평균 위험지수 변화, 가장 큰 위험 감소, 개선된 시군구 수)
6. TOP 10 개선 효과 시군구 리스트

**공통**
7. About 페이지 (데이터 출처, 방법론, 한계, 가점 신청 항목)

**대회 제출물**
- 시제품 (Vercel 프론트 URL + Railway 백엔드 API)
- 기획서 (대회 양식, **최대 3장**)
- 데모 영상 (1~2분)

## 기술 스택 (풀스택)

**프론트엔드**
- Next.js 14 (App Router) + TypeScript
- Tailwind CSS + shadcn/ui (선택)
- 지도: MapLibre GL JS + deck.gl (오픈소스, 토큰 불필요)
- 차트: Recharts
- 배포: Vercel

**백엔드**
- FastAPI (Python 3.11+)
- 기존 ML 코드 (`backend/src/`) 그대로 wrap — `train.py`, `inference.py`, `shap_explain.py`
- 정적 데이터 (`grid_features.parquet`) 메모리 로드
- 모델 (`xgb_risk_model.pkl`) 메모리 로드
- 배포: Railway (Python 무료티어, GitHub 연동)

**디렉토리 구조**
```

│   ├── api/                # FastAPI endpoints (작성 예정)
```

## 가점 신청 항목 (대회 공식 양식 — Q13)

가점 항목은 **심사위원단 판단**에 따라 결과 결정. "확보"가 아니라 **신청**이라는 점 명시.

| 가점 항목 | 배점 | BlindZone 신청 여부 | 비고 |
|---|---|---|---|
| 가명정보 결합 | 5 | X | |
| **데이터 융합** (주관기관별 데이터 융합) | 5 | ✅ 신청 | 사고(TAAS) × 응급의료기관 × 119 × 시군구 행정경계 = 4개 |
| 안심구역 활용 | 5 | X | |
| **AI 분석도구** | 5 | ✅ 신청 | XGBoost + SHAP. 학습·평가 로그 증빙 가능 |
| **AI 학습도구** | 5 | ✅ 신청 (단, 단순 검색 판단 시 제외 위험) | Claude Code 활용. **증빙자료 제출 필요** — 코드 작성 기여 로그·스크린샷 준비 |

→ 최대 신청 가능: **15점**. 실제 부여 여부는 심사위원단 결정.

## 1차 심사 제출물 양식 (Q9)

- 기획서 **최대 3장** — 한글 양식 (`경진대회_참가서류.zip` 첨부)
- 제품/서비스 분야: **앱·웹 URL** 또는 **프로토타입(시제품)** 반드시 첨부
- 기타 자료 별도 첨부 가능
- 참가신청서는 경진대회 페이지에서 다운로드

→ BlindZone 제출:
- 기획서 (3장, 한글 양식)
- Vercel 데모 URL + Railway API URL
- 데모 영상 (1~2분, 별도 첨부)
- AI 학습도구 사용 증빙 (Claude Code 사용 기록)

## 정직성 원칙 (절대)

산출물(About / README / 기획서 / 발표) 작성 시 다음 8가지 위험 표현을 **반드시** 정확하게:

| 위험 표현 | 정정 |
|---|---|
| "사고 위험을 예측한다" | "공개 데이터로 정의한 잠재 위험지수를 계산·설명한다" |
| "응급 도착시간" | "응급의료기관 거리 기반 추정 접근시간 (60km/h 가정)" |
| "정책 효과를 예측한다" | "거리 기반 접근성 지표 변화 민감도 분석" |
| "연간 사고 건수" | "TAAS 사고다발지역 데이터 내 사고 건수" |
| "기존 일반인용 서비스 부재" | "기존 사고다발지도는 빈도 중심, 본 프로젝트는 응급 접근성 결합 탐색" |
| "주관기관 4종 결합" | "공공데이터 4종 결합" |
| "AI 가점 15점 확보" | "가점 신청 (부여는 심사위원단 판단)" |
| "R² > 0.5 = 사고 예측 성능" | "정의된 잠재 위험지수에 대한 surrogate model 재현도" |
